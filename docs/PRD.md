# PRD：工资结算表生成器

## 产品概述

一款 Windows 桌面应用，供教师录入上课记录，自动计算薪酬并导出为工资结算表 Excel 文件。

## 目标用户

- **当前**：个人使用（每月底集中录入并导出）
- **中期**：公司同事（不同使用习惯，可能需要逐日录入）
- **远期**：可能扩展为微信小程序 + 管理系统

## 核心功能

### 1. 上课记录录入
- 输入/选择：教师姓名、日期、开始时间、课时数（浮点数）、班级类型（下拉菜单）
- 点击"添加"存入当前列表

### 2. 预览列表
- 表格形式展示所有已录入记录
- 自动显示：星期、时薪（按班级类型）、薪酬（课时数 × 时薪）
- 支持删除记录

### 3. Excel 导出
- 用户选择保存路径和文件名
- 生成格式参考 `screenshot_original-effect.png` 及参考 Excel 文件
- 同班级行自动着色区分（班级类型 + 星期 + 开始时间 三项完全一致视为同班级）

## 技术方案

| 模块 | 技术 | 说明 |
|---|---|---|
| GUI | PySide6 | LGPL 协议，组件丰富 |
| Excel | openpyxl | Python Excel 读写标准库 |
| 存储 | JSON | 本地文件，轻量零依赖 |
| 语言 | Python 3.10+ | |

## 数据模型

```python
Record(date, start_time, hours, class_type, note)
MonthData(teacher_name, records: list[Record])
```

## 班级费率配置

| 班级类型 | 时薪（元/时） | 状态 |
|---|---|---|
| 1对1 | 90 | 预留 |
| 1对2 | 100 | 在用 |
| 1对3 | 100 | 在用 |
| 1对4 | 110 | 在用 |
| 1对5 | 120 | 在用 |

费率配置放在独立模块中，支持手动新增/修改。

## Excel 导出规范

详见 [docs/2026-05-31-需求讨论记录.md](docs/2026-05-31-需求讨论记录.md#excel-格式规范基于参考文件解析)

简要：
- A~K 列，第1行标题行（合并单元格）
- 第2行显示教师姓名
- 第3行表头
- 第4行起数据
- 末行合计
- 薪酬列使用 Excel 公式而非硬编码数值
- 同班级同背景色（班级类型 + 星期 + 开始时间 三项一致视为同班级，三项中任一不同即换色）

## 非功能需求

- 架构分层：数据模型 → 存储层 → UI 层，模型层不引用框架代码
- 为未来迁移 uniapp 留余地
- 不引入数据库，不引入不必要的抽象
