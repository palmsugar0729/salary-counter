## 遇到的问题

### Edit 工具 + 中文引号编码破坏
使用 Edit 工具替换含中文弯引号 `""` (U+201C/U+201D) 的代码时，写入文件后字符编码损坏。尝试替换 Unicode 转义 `“` 但 Python 脚本自身解释掉了。最终改用 `「」` 角括号替代显示层的弯引号，避开编码问题。

### f-string 中 `\n` 变字面换行
Python 脚本写文件时，字符串中的 `\n` 被写成物理换行，导致 f-string 跨行断开。用 Edit 工具将断开的 f-string 修回单行。

**教训**：含特殊 Unicode 字符文件的复杂编辑，用脚本整块替换比多次 Edit 调用更可靠；或用 Write 直接写整个文件更安全。

## GitHub Actions 多平台 CI

用户没有 Mac 但想打 Mac 版。配置了 `.github/workflows/build.yml`，推送 `v*` tag 自动打包 Windows / macOS / Linux 三个平台。

### 首次构建失败 + 修复

Linux 作业在 `apt-get install libgl1-mesa-glx` 失败，因为 Ubuntu 24.04 已移除该包。同时 `fail-fast` 默认为 true，Linux 一失败就把 macOS 和 Windows 全取消了。

**修复**（`fe6e67c`）：
- `libgl1-mesa-glx` → `libgl1 libopengl0`（适配 Ubuntu 24.04）
- `strategy.fail-fast: false`（各平台独立，互不影响）

修完后删除并重建 `v0.1.1` tag 重新触发，三个平台全部通过。
